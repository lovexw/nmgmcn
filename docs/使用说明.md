## 项目概述

- 提供便捷的联系渠道

## 技术架构

### 技术栈详解

- **Vue 3**：采用最新的Composition API和`<script setup>`语法，提高代码可读性和维护性
- **Vite**：下一代前端构建工具，提供极速的开发体验
- **Element Plus**：基于Vue 3的组件库，提供丰富的UI组件
- **Vue Router**：Vue官方路由管理器，实现单页应用的路由功能
- **Font Awesome**：提供丰富的图标资源

## 开发指南

### 环境配置

1. **安装Node.js和npm**
   - 推荐使用Node.js 16.0+版本
   - npm 7.0+或yarn 1.22+

2. **编辑器配置**
   - 推荐使用Visual Studio Code
   - 安装Volar插件以获得更好的Vue 3支持
   - 安装ESLint和Prettier插件以保持代码风格一致

### 开发规范

1. **命名规范**
   - 组件名：使用PascalCase（如NavBar.vue）
   - 变量名：使用camelCase（如activeCategory）
   - 常量：使用UPPER_SNAKE_CASE（如MAX_COUNT）

2. **代码风格**
   - 使用2个空格缩进
   - 每行代码不超过100个字符
   - 组件属性按字母顺序排列

3. **组件开发**
   - 使用Composition API和`<script setup>`语法
   - 组件结构遵循template -> script -> style的顺序
   - 页面样式采用scoped CSS，避免样式污染

4. **注释规范**
   - 为复杂的业务逻辑添加注释
   - 组件的props和emits应有清晰的注释说明
   - 使用JSDoc风格的注释

## 部署指南

### 开发环境部署

1. 克隆项目到本地
```bash
git clone [项目仓库地址]
cd nmgmcn
```

2. 安装依赖
```bash
npm install
# 或
yarn install
```

3. 启动开发服务器
```bash
npm run dev
# 或
yarn dev
```

4. 在浏览器中访问 http://localhost:3000 (或控制台显示的URL)

### 生产环境部署

1. 构建生产版本
```bash
npm run build
# 或
yarn build
```

2. 部署dist目录到Web服务器
   - 使用Nginx作为Web服务器（推荐）
   - 配置示例：

```nginx
server {
    listen 80;
    server_name example.com;

    root /path/to/dist;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }
}
```

3. 配置HTTPS（推荐）
   - 使用Let's Encrypt获取免费SSL证书
   - 配置Nginx支持HTTPS

4. CDN加速（可选）
   - 将静态资源部署到CDN以提高访问速度
   - 修改vite.config.js中的base配置指向CDN地址

## 常见问题解答

### 1. 开发环境常见问题

**Q: 启动开发服务器时报错"Port 3000 is already in use"**

A: 端口3000已被占用，可以使用以下命令指定其他端口：
```bash
npm run dev -- --port 3001
```

**Q: 安装依赖时报错**

A: 尝试清除npm缓存后重新安装：
```bash
npm cache clean --force
npm install
```

### 2. 构建部署常见问题

**Q: 构建后页面空白或资源404**

A: 检查vite.config.js中的base配置是否正确，确保与部署路径一致。

**Q: 部署后页面刷新出现404**

A: 这是因为使用了Vue Router的history模式，需要在服务器配置URL重写规则，将所有请求重定向到index.html。

### 3. 功能开发常见问题

**Q: 如何添加新页面？**

A: 
1. 在src/views目录下创建新的Vue组件
2. 在src/router/index.js中添加对应的路由配置
3. 在NavBar.vue中添加导航链接

**Q: 如何修改网站主题色？**

A: 主题色定义在全局CSS变量中，可以在App.vue的style标签中修改。
